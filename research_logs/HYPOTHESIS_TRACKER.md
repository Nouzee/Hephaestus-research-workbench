## 猜想 1：市场微观结构可以用字典学习分解为少数"模式"，且模式可以预测未来

**初始猜想**（BTC 阶段）：
> 把 L2 订单簿的 10 档深度用 NMF 压缩成 6-8 个市场模式，模式之间的转移规律可以预测 mid-price 变动。

**具体做法**：
- 输入矩阵 X: T × 20（每行 10 档 bid qty + 10 档 ask qty）
- NMF 分解：X ≈ W × H，其中 W 是 T×k 的权重矩阵，H 是 k×20 的模式矩阵
- k=6 时，分解出 6 个市场原子（Market Atoms）：

| 原子 | 订单簿形态 | 经济学含义 |
|---|---|---|
| **Atom 1** | bid 侧深度集中（前 3 档厚），ask 侧薄 | 买盘支撑——有人在 bid 侧堆积大单 |
| **Atom 2** | ask 侧深度集中，bid 侧薄 | 卖盘压力——卖方流动性占优 |
| **Atom 3** | bid/ask 均衡，各档深度均匀且厚 | 稳定流动性——做市商活跃，spread 窄 |
| **Atom 4** | bid/ask 都薄，深度向远档塌陷 | 流动性撤离——没人愿意在近档挂单 |
| **Atom 5** | spread 宽，近档深度极薄，远档略厚 | 高波动/新闻冲击——做市商撤退，spread 扩大 |
| **Atom 6** | bid/ask 深度不对称 + spread 动态变化 | 方向性压力——一侧被持续消耗，spread 在变化 |

**阶段结论**：
> NMF 成功将订单簿分解为 6 个可解释的市场原子。每种原子对应一种明确的微观结构状态。这验证了"订单簿存在低维结构"的假设。

**后续推翻**：
- 构建了 6×6 的转移概率矩阵 A（模式动力学），但 Granger 因果检验全部不显著：过去的模式权重不能预测未来的 mid-price 变动
- R² 天花板 = 8%：即使最优基向量组合，也只能解释 mid-price 变动的 8%，剩余 92% 是噪声
- 做了 MCT（可控性测试）：假设能精确控制某个原子的权重，能否推动市场？答案是不能——单变量干预被系统吸收，其他维度自动补偿
- 在真实订单簿上模拟连续被动做市：Sharpe ≈ -40，adverse selection 完全主导
- 做了 MSDP（最小尺度发现）：市场存在稳定不动点，无论怎么扰动都会回归——没有可剥削的漂移

**最终结论**：
> 字典学习可以**描述**市场当前状态，但不能**预测**市场未来方向。统计模式 ≠ 可交易 edge。这个教训直接塑造了后续所有研究：**不再问"市场会怎样"，而是问"在当前状态下，被动做市是否盈利"。**

## 猜想 2：A 股可能比 BTC 更有结构性，存在可交易的微观结构 edge

**初始猜想**（A 股转向）：
> A 股有基本面锚定、交易时段（4 小时 vs 24 小时）、涨跌停限制（±10%），其微观结构可能比永续合约更容易建模。

**具体做法**：
- 标的：000333 美的集团，59 个交易日 L2 数据（十档订单簿 + 逐笔成交）
- 从每 tick 订单簿快照提取 16 个微观结构特征：

| 维度 | 特征 | 计算方式 |
|---|---|---|
| 价格 | Spread | OfferPrice1 - BidPrice1 |
| 价格 | Mid-price return | Δ(mid) / mid |
| 深度 | Bid total depth | sum(BidQty1→5) |
| 深度 | Ask total depth | sum(OfferQty1→5) |
| 平衡 | OBI (Order Book Imbalance) | (BidDepth-AskDepth)/(BidDepth+AskDepth) |
| 平衡 | Depth concentration | Qty1 / sum(Qty1→5) |
| 波动 | Spread volatility | std(spread) over window |
| 波动 | Mid volatility | std(mid return) over window |
| 流量 | Trade count | 窗口内成交笔数 |
| 流量 | Trade size mean/std | 成交量的均值/标准差 |
| 流量 | Direction imbalance | (BuyVol-SellVol)/(BuyVol+SellVol) |
| 流量 | Trade intensity | 成交笔数/时间 |
| 形态 | Bid depth skew | Qty1 vs Qty2→5 的分布偏度 |
| 形态 | Ask depth skew | 同上，ask 侧 |
| 形态 | Spread- depth interaction | spread × depth 的交叉项 |
| 形态 | Price-depth correlation | mid 变动与深度变动的相关性 |

- 每 100 tick 为一个窗口，对每个特征计算窗口内均值/标准差
- 前 20 天数据训练 KMeans(k=8)，得到 8 个 regime
- Tox = Spread / TotalDepth，按训练集分位（p30, p70）分为三档
- 时间段：早盘（0-30%）、中盘（30-70%）、尾盘（70-100%）
- 状态编码：R{r}_q{tq}_T{td}，共 8×3×3 = 216 个可能状态

**8 个 Regime 的具体特征**：

| Regime | 聚类中心特征 | 经济学标注 |
|---|---|---|
| **R0** | 深度极薄，spread 宽，vol 高 | 流动性崩溃 / 恐慌 |
| **R1** | ask 深度集中（bid 薄），OBI 偏负 | 卖盘压力主导 |
| **R2** | 深度极厚，spread 适中，vol 低 | 深度流动性（看似安全，实为陷阱） |
| **R3** | trade count 极高，成交密集 | 交易浪潮 |
| **R4** | bid 深度集中（ask 薄），OBI 偏正 | 买盘支撑 |
| **R5** | spread 中等，OBI 持续偏负，vol 适中 | 压力吸引子（持续性卖压，自转移 93.9%） |
| **R6** | 主动买盘方向 + spread 宽 | 买方主导的宽 spread 状态 |
| **R7** | 主动卖盘方向，ask 深度薄 | 卖方主导 |

**阶段结论**：
> A 股微观结构确实比 BTC 更可分类。8 个 regime 在经济学上有明确含义，Tox 是区分盈利/亏损的关键变量。但方向是反的——宽 spread 反而更安全。

## 猜想 3：低 Toxicity（窄 spread + 厚深度）的流动性好的状态更安全

**初始猜想**（传统观念）：
> spread 窄 = 交易成本低 = 容易赚钱。深度厚 = 流动性好 = 成交快。

**具体数据发现**：
- 在 15 个滚动验证窗口中，每个窗口独立计算 tox 分位（只用训练集数据校准 p30/p70）
- **相变点位于 30% tox 分位，15/15 窗口一致出现**
- Tox < 30%（窄 spread + 厚深度）：E[PnL/fill] ≈ 0 或负
- Tox > 30%（宽 spread + 薄深度）：E[PnL/fill] > 0，且随 tox 单调递增
- 做反证测试：shuffle tox → edge 消失；shuffle state → edge 消失

**经济学解释**：
- 低 tox：很多人同时挂单在 best bid/ask → 你的订单排在队列深处 → 成交慢 → 即使成交，也是因为有人用大市价单扫掉了前面所有深度（这是强方向信号）→ adverse selection 吃掉你的 spread
- 高 tox：做市商少，队列短 → 你的订单快速到达队首 → spread 宽 → 即使在不利选择中成交了一两笔，宽 spread 的均值回归也能保护整体仓位

**阶段结论**：
> **毒性反转（Toxicity Inversion）**：被动做市的视角下，"安全"的低 tox 状态是亏损区（竞争激烈 + adverse selection），"危险"的高 tox 状态是盈利区（竞争少 + mean reversion 保护）。传统市场微观结构文献的"toxicity"概念在被动做市中恰好方向相反。

## 猜想 4：状态越多，参与机会越多，总 PnL 越高

**初始猜想**：
> 用连续权重按状态质量分配仓位（好的多投，差的少投），比简单的"参与/不参与"更优。

**具体做法**：
- 对比了两套方案：
  - 方案 A：二值过滤——在 CORE 状态全仓参与，非 CORE 状态完全不参与
  - 方案 B：连续加权——按状态的 E[PnL/fill] 作为权重，连续调整仓位
- 在相同的执行模型和回测框架下对比

**具体数据**：
- 二值过滤的 PnL 比连续加权高 **+111%**
- 解释：中间状态（tox 在 25-35% 之间的"灰色地带"）在连续加权中会被分配少量仓位，但这些少量仓位的 adverse selection 成本不成比例地高——你参与了一点点，但每单位参与的损失比完全不参与更大
- 相变是**尖锐的**（sharp phase transition），不是渐进的

**阶段结论**：
> 在相变点附近，模糊是有害的。被动做市的盈亏函数在 30% tox 处有一个陡峭的拐点——拐点以下几乎不赚钱，拐点以上明确赚钱。试图"平滑过渡"只会把拐点以下的劣质状态也带进来。

## 猜想 5：成交概率可以近似为固定常数

**初始猜想**：
> 每个 tick 挂的单以固定概率 P=0.30 成交（Bernoulli fill model），不需要模拟队列动态。

**具体做法（EVL 队列模拟器）**：
- 在每 tick 以 100 股在 best bid/ask 挂单
- 记录当前队列位置 = 当前该价位的总挂单量 + 我们的 100 股
- 向前遍历后续 tick（最多 100 tick）：
  - 如果有人以我们的价格成交 → 队列位置减去成交量
  - 如果该价位深度缩减（撤单） → 队列位置减去缩减量
  - 如果队列位置 ≤ 0 且有成交 → **触发 fill**
- 记录 fill 后 100 tick 的 mid 变动（markout），按交易方向符号化

**具体数据（10 天模拟）**：

| 指标 | Bernoulli 假设 | EVL 真实值 | 误差 |
|---|---|---|---|
| 平均 fill rate | 30.0% | **82.4%** | 2.75x 低估 |
| R0_q2_T0 fill rate | 30%（假设） | 84.0% | — |
| R6_q2_T0 fill rate | 30%（假设） | 80.7% | — |
| **R2_q2_T0 fill rate** | 30%（假设） | **0.7%** | **43x 高估** |
| R2 markout（fill 后 100tick） | 未建模 | **-49 bps** | 灾难级 |
| 平均队列等待 | 未建模 | 25-33 ticks | — |
| R2 A/E ratio | 未建模 | **4.42** | 每赚 1 块亏 4.42 块 |

**R2 为什么是执行陷阱**：
- 深度极厚（看起来流动性好、安全）
- Spread 较宽（看起来利润高——实际上 STATE_EV 显示 R2 的 spread capture 是 654 raw units，所有状态中最高）
- 但队列排在你前面的深度太多，你永远到不了队首
- 即使成交，也是因为有巨大的方向性订单扫掉了所有深度——此时 mid 已经对你不利地移动了 49 bps

**阶段结论**：
> 原来的 15 个 CORE 状态中，R2 必须被排除。ECORE（执行感知 CORE）从 15 个状态缩减/重组为 **21 个状态**——不是简单减少，而是用执行数据重新校准。R2 被永久排除。Pareto CORE（基于多目标优化的方法）只保留了 3 个状态，过于严格。**执行层不是事后补充——它就是策略的核心筛选机制。**

## 猜想 6：EBACKTEST 的 Sharpe 7.82 / 0 回撤 / 100% 胜率说明策略很强

**初始猜想**：
> 首次集成回测结果极好，策略可能真的能赚钱。

**原始 EBACKTEST 的 PnL 计算（有 bug 的版本）**：
```python
# 原始代码 —— 每一行都是一个错误
pnl += n_fills_binom * (sp_capture - markout_raw)
# 问题：把每笔成交当作完成的往返交易
# 没有 bid/ask 分边
# 没有库存跟踪
# 没有 MTM
```

**BIA 审计发现的具体三个 bug**：

**Bug 1 — 无库存跟踪**：每笔 fill 直接按 `sp_capture - markout` 计入 PnL，不区分是买还是卖。实际上被动做市的利润来自**买入和卖出的价差**——你必须先买才能卖。原始代码把每笔成交都当成一次完整的低买高卖。

**Bug 2 — 无 MTM**：equity 从未计算 `equity = cash + inventory × mid_price`。没有未实现盈亏的概念。这就是为什么 Max DD = 0——根本没有东西可以回撤，代码只记录了已实现的 spread capture。

**Bug 3 — 无 bid/ask 拆分**：不知道当前持有多少净头寸。无法判断是做多还是做空。无法检测库存漂移。

**修正后的数据**：

| 指标 | 原始（Bug） | BIA 修正后 |
|---|---|---|
| ECORE Total PnL | +100M | +13.1B（注意这是 raw units，实际 CNY / 10,000） |
| Sharpe | 7.82 | 0.61 |
| Max DD | 0 | 4.3B |
| 胜率 | 100% | 65% |
| 亏损窗口 | 0 | 31,791 个 |

**阶段结论**：
> **回测会计不是实现细节。三个 bug 导致了数量级的偏差。** 修正后 edge 仍然为正（PnL/quote ECORE = 2.66x vs Always-On），但幅度远小于原始估计。核心教训：**默认假设"好结果 = bug"，主动设计反证实验。**

## 猜想 7：库存会自然均值回归，不需要主动管理

**初始猜想**：
> bid 和 ask 成交概率相同，库存会在零附近波动，不需要干预。

**具体数据**：
- 对称报价下的库存自相关 = **1.000**（完美单位根）
- 半衰期 = **693 个窗口**（≈ 69,300 tick）
- 平均持仓时长 = 2,373 个窗口（几乎永远不平仓）
- 库存可以从 +45,000 漂移到 -40,000
- MTM 波动（inventory × Δmid_change）占 equity 方差的主导部分
- 自然回归状态数量 = 0——没有任何状态能使库存系统性回归

**阶段结论**：
> 被动做市不存在自然库存均值回归。对称报价产生的库存是一个纯粹的随机游走。**必须主动控制。**

## 猜想 8：偏斜报价会损害执行 edge

**初始猜想**：
> 不对称报价（偏斜 bid 或 ask）会减少盈利成交，降低总 PnL。

**具体做法（IECORE 三级偏斜引擎）**：

**Level 1 — Size Skew**（始终在线）：
```
inv_frac = inventory / 50,000

Long 时（inv > 0）：
  bid_mult = 1.0 - 0.5 × inv_frac   # 减少买盘
  ask_mult = 1.0 + 0.5 × inv_frac   # 增加卖盘
Short 时（inv < 0）：反向操作
```

**Level 2 — Price Skew**（始终在线）：
```
Long：bid 报价远离 mid（更差），ask 报价靠近 mid（更好）
      → 减少买入成交概率，增加卖出成交概率
```

**Level 3 — Suppression**（极端时激活）：
```
|inv| > 35,000（70% 上限）：同侧 fill 概率 × 0.3
|inv| > 45,000（90% 上限）：同侧 fill 概率 = 0，完全停止
```

**State-Conditioned 偏斜强度**：
- 高 A/E 状态（R6，A/E=0.45）：偏斜强度 = 0.5 × (1-0.45) = 0.28（弱偏斜，保护执行 edge）
- 低 A/E 状态（R1，A/E=0.08）：偏斜强度 = 0.5 × (1-0.08) = 0.46（强偏斜，安全空间大）

**具体数据**：

| 控制器 | 总 PnL | Sharpe | Max DD | 库存标准差 | 半衰期 |
|---|---|---|---|---|---|
| NoSkew | +14.2B | 1.90 | 1.72B | 13,939 | 693w |
| Mild (0.3) | +15.1B | 14.90 | 192M | 1,527 | 76w |
| **Moderate (0.5)** | **+15.2B** | **22.35** | **176M** | **1,114** | **40w** |
| Aggressive (0.8) | +15.1B | 30.83 | 123M | 1,028 | 33w |

**阶段结论**：
> 库存控制不创造 alpha，但也不破坏 alpha。PnL 在所有控制器下几乎相同（~15.1B），但风险指标显著改善（DD -90%，库存 std -92%）。偏斜引擎**降低噪音，让执行 edge 不被 MTM 波动淹没**。

## 猜想 9：IECORE 的 Sharpe 22-30 被频率假象夸大了

**初始猜想**：
> 90,478 个窗口 × sqrt(90K) ≈ 300 倍年化放大 → 真实 Sharpe 应该远低于 22。

**具体数据（多频率 Sharpe 对比）**：

| 方法 | Sharpe | 说明 |
|---|---|---|
| IECORE 方法（窗口 × sqrt(N)） | **20.43** | 最保守的估计 |
| 窗口日级年化 | 42.23 | sqrt(252 × 144) |
| **日聚合，年化** | **33.66** | 标准方法 |
| 日聚合，AR(1) 修正 | **19.71** | 修正自相关后 |
| 月聚合，年化 | 227.66 | N=2，无意义 |
| 非重叠 5 日 | 21.72 | N=7，无重叠 |

**为什么 IECORE 的 Sharpe 反而是最低的**：
- 窗口级 PnL 有负自相关（AR(1) = -0.420）——库存控制的偏斜在窗口间产生均值回归效应
- 负自相关意味着窗口间方差在日聚合时被部分抵消 → 日级 std 低于独立假设预期 → 日级 Sharpe 反而高于窗口级 Sharpe
- **sqrt(N) 并没有夸大 Sharpe——它在这个 case 里反而是保守的**

**但真正的红旗**：
- 35 个交易日，**0 个亏损日**（每天都是正的）
- 日级 AR(1) = 0.489 → 有效样本仅 12 天
- 5 日测试窗口 AR(1) = 0.803 → 有效 N = 1
- BootStrap 95% CI：[28.09, 44.25]——区间虽窄但仅基于 35 天

**阶段结论**：
> Sharpe 没有被频率假象夸大。但 35 天 0 亏损日极度可疑——样本太小，无法可靠估计真实 Sharpe。需要更长的测试期或跨资产验证。

## 猜想 10：Edge 可能来自方向暴露

**初始猜想**：
> SA 发现 MTM 占 96% 方差。如果研究期内 mid 趋势性上涨 + 持有正库存 = 不需要执行 skill 也能赚钱。

**具体做法（DNA 分解器）**：
```
Total PnL_t      = Equity_t - Equity_{t-1}
Inventory PnL_t  = inventory_{t-1} × (mid_t - mid_{t-1})    ← 方向性贡献
Execution PnL_t  = Total PnL_t - Inventory PnL_t            ← 纯执行贡献
```

然后跑窗口级回归：PnL_t = α + β × Δmid_t + ε

**具体数据**：

| 分解 | Symmetric（无控制） | Moderate（有控制） |
|---|---|---|
| Total PnL | +13.62B | +15.05B |
| Execution PnL | **+15.09B (+111%)** | **+15.10B (+100%)** |
| Inventory PnL | **-1.47B (-11%)** | **-0.05B (-0%)** |

- Symmetric 下库存 PnL 是**负的**（-11%，-1.47B）——方向漂移在消耗利润
- 原因：无控制时系统偏空（mean inventory = -7,076），而 mid 在研究期内上涨 → 空头在牛市中亏钱
- Moderate 下 Beta ≈ 0，R² with mid ≈ 2.2%
- Long/short 完全对称：t-stat = -1.26，不显著
- 三个子周期（Early/Mid/Late）执行 PnL 稳定（+4.15B / +3.81B / +7.14B），与累计 Δmid（+5,650 / +30,000 / +26,900）不相关

**阶段结论**：
> **Edge 是纯执行的——来自 spread capture 减去 adverse selection。** 方向暴露不是利润来源，而是利润的拖累（无控制时吞噬 11% 的 PnL）。库存控制消除的不是"优势"，而是"劣势"。

## 猜想 11：OOS 条件下 Edge 会衰退

**初始猜想**：
> 所有参数（ECORE 定义、tox 阈值、skew 强度）在前 40 天训练集上确定，后 19 天可能分布不同。

**具体做法（FOOS 冻结验证）**：
- 硬切分：前 40 天 = 训练，后 19 天 = 冻结 OOS
- OOS 数据只看一次，不调参
- 迁移完整 Hephaestus 管线（ECORE + ETE + IECORE Moderate）

**具体数据**：

| 指标 | In-Sample（40天） | OOS（19天） | 变化 |
|---|---|---|---|

| Execution PnL | +4.55B | +5.12B | +12% |
| Daily Sharpe (total) | 37.57 | 44.07 | +17% |

## 猜想 12：A 股零售费率下，执行 edge 可以覆盖交易成本

**初始猜想**：
> Spread capture 虽然小，但成交量大，总 PnL 应该能覆盖费用。

**具体费用结构（A 股真实费率）**：

| 费用项 | 买方 | 卖方 |
|---|---|---|
| 佣金 | 2.5 bps | 2.5 bps |
| 交易所规费 | 0.5 bps | 0.5 bps |
| 过户费 | 0.2 bps | 0.2 bps |
| **印花税** | — | **5.0 bps** |
| **合计** | **3.2 bps** | **8.2 bps** |

**关键计算**（000333，mid ≈ 80 CNY，100 股/笔）：
```
每笔成交金额（notional）= 80 × 100 = 8,000 CNY
每笔费用（买方）= 8,000 × 0.00032 = 2.56 CNY
每笔费用（卖方）= 8,000 × 0.00082 = 6.56 CNY
平均每笔费用 = 4.56 CNY

每笔 spread 捕获（½ spread, ~2.5 bps）= 80 × 100 × 0.00025 / 2 ≈ 1.0 CNY
费用/spread 比 = 4.56 / 1.0 = 4.56x
```

**测试了 16 个摩擦场景**：

| 场景 | 000333 PnL | 601899 PnL | 状态 |
|---|---|---|---|
| 零摩擦基线 | +5.78B | — | OK |
| **零售费率 + 1bp 滑点** | **-39.0B** | — | **死** |
| Fill 衰减 30% | -11.7B | — | 死 |
| Adverse ×3 | -70.0B | — | 死 |
| Spread ×0.5 | -43.8B | — | 死 |
| 重度 Impact 5x | -160.3B | — | 死 |
| 综合极端压力 | -53.8B | — | 死 |

**16/16 场景全部失败。0.5x 仓位规模已经亏损。**

**阶段结论**：
> A 股零售费率下，100 股级别的被动做市在经济上不可行。**这不是策略问题——是市场结构问题。** 当前 A 股的 spread 水平（2-7 bps）无法覆盖当前的费率水平（每笔 3.2-8.2 bps）。

## 猜想 13：所有 A 股都一样，换标的不会改变结论

**初始猜想**：
> 000333 的结论应该泛化到其他 A 股。

**具体数据（601899 紫金矿业，mid ≈ 30 CNY）**：

| 对比维度 | 000333（美的） | 601899（紫金） | 比率 |
|---|---|---|---|
| 平均股价 | ~80 CNY | ~30 CNY | 0.38x |
| Spread（bps of mid） | 2.5 bps | **6.6 bps** | **2.63x** |
| 每笔费用（买方） | 2.56 CNY | **0.97 CNY** | 0.38x |
| 每笔费用（卖方） | 6.56 CNY | **2.49 CNY** | 0.38x |
| **Edge 厚度（ETR）** | **0.13** | **0.35** | **2.63x** |
| ECORE 占比 | 7.8% | **23.3%** | 3.0x |
| R6 attractor | 主导 ECORE | **几乎不存在** | 0.3% vs 0.0% |

**为什么 601899 更好**：
- 股价更低（30 vs 80 CNY）→ 手续费与成交金额成正比 → 每笔费用同比例降低
- Spread 在 tick 单位下不完全等比缩放 → bps spread 更宽（6.6 vs 2.5）
- 两者叠加 → **ETR 高出 2.63 倍**
- ECORE 占比 3 倍 → 更多可交易时间

**但注意**：regime 结构完全不同。R6 attractor 在 601899 上消失（17/537K 窗口），Top ECORE 变成了 R0/R1/R3。**不同股票的市场微观结构是不同的——不能直接复制状态定义。**

**阶段结论**：
> 股价是影响经济可行性的核心变量。低价股天然更适合这种 execution edge。但每个股票的 regime 结构不同，需要独立校准状态分类器。

## 猜想 14：Edge 在任何费率下都不可行

**初始猜想**：
> 零售费率 16/16 全死了，可能什么费率都不行。

**具体数据（VF 可行性边界分析 — 7 种费率场景 × 2 只股票）**：

| 费率场景 | 000333 ETR | 601899 ETR | 判定 |
|---|---|---|---|
| 零售（佣金 2.5bp + 印花税 5bp） | 0.13 | 0.35 | 均不可行 |
| 低佣金（1bp） | 0.17 | 0.45 | 均不可行 |
| 机构（0.5bp） | 0.23 | 0.60 | 均不可行 |
| 免印花税（保持零售佣金） | 0.21 | 0.56 | 均不可行 |
| **做市商计划（0.1bp + 免印花税）** | **1.19** | **3.14** | **两者均可行** |

**Spread 边界分析**：
- 000333 需要 **7.5x spread**（18.6 bps）才能在零售费率下盈亏平衡
- 601899 需要 **2.8x spread**（18.6 bps）才能在零售费率下盈亏平衡
- 601899 在做市商计划下，什么都不用改，**ETR = 3.14**（安全边际充足）

**股价敏感性分析（恒定 raw spread ≈ 199）**：

| 股价 | Spread (bps) | ETR | 零售费率下可行？ |
|---|---|---|---|
| 10 CNY | 19.9 bps | **1.07** | **是** |
| 20 CNY | 10.0 bps | 0.54 | 否 |
| 30 CNY | 6.6 bps | 0.36 | 否 |
| 80 CNY | 2.5 bps | 0.13 | 否 |

**市场选择规则（5 维加权评分）**：
1. 更低的股价（权重 15%）→ 手续费用更低
2. 更宽的 spread bps（权重 30%）→ 更多捕获
3. 更高的 q2 占比（权重 20%）→ 更多报价机会
4. 更低的 A/E ratio（权重 20%）→ 更少 edge 侵蚀
5. 更高的 fill 概率（权重 15%）→ 更多成交

**理想目标画像**：股价 < 30 CNY，spread > 5 bps，q2 > 20%，A/E < 0.25，fill > 80%。

**阶段结论**：
> **Edge 不是不存在——它需要正确的市场条件 + 费率结构的组合。** 做市商计划（免印花税 + 极低佣金）是最直接的可行路径。在零售费率下，10 CNY 以下的低价股 + 宽 spread 也可行。**问题从"能不能赚钱"变成了"在什么条件下能赚钱"。**
